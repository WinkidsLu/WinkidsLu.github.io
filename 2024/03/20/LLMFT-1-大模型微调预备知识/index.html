LLMFT(1):大模型微调预备知识

发表于2024-03-20|更新于2024-03-22|LLM

0. 大模型微调概述

为什么要微调？

  1. 预训练成本高（65B需要780GB显存）
  2. 提示工程有天花板（任何提示工程依托于基础模型，token存在上限与推理成本；生成token与需要的成本呈平方关系）
    1. 构建模板（Template Construction）：通过人工定义、自动搜索、文本生成等方法，生成与给定句子相关的一个含有[MASK]的模板，例如It was [MASK].，拼接到原始文本中获得prompt-tuning的输入: [CLS] I like the Disney film very much. [SEP] It was [MASK]. [SEP] 并复用预训练好的MLM分类器（在huggingface中为BertForMaskedLM），即可直接得到 [MASK]预测的各个token的概率分布；
    2. 标签词映射（Label Word Verbalizer） ：因为 [MASK] 部分我们只对部分词感兴趣，因此需要建立一个映射关系。例如如果 [MASK]预测的词是“great”，则认为是positive类，如果是“terrible”，则认为是negative类。
    3. 训练 ：根据Verbalizer，则可以获得指定label word的预测概率分布，并采用交叉信息熵进行训练。此时因为只对预训练好的MLM head进行微调，所以避免了过拟合问题
  3. 基础模型缺少特定领域数据（某些大模型中文语料少）
  4. 数据安全与隐私
  5. 个性化服务需要私有化的微调大模型

微调技术路线：

  1. 全量微调（Full Fine-Tune，FFT）—— 加载所有模型参数到GPU，只训练部分参数
  2. 高效微调（Parameter-Efficient Fine-Tune，PEFT）
    • 有监督微调（Supervised Fine-Tune，SFT）—— 基于标注、有监督的
    • 基于人类反馈的强化学习（RLHF）—— 人类价值观对齐，优点：训练质量高/缺点：人类介入，效率不高
    • 基于AI反馈的强化学习（RLAIF）

全量微调的问题：

  1. 训练成本高：仅有少部分公司可以实现
  2. 灾难性遗忘：特定领域数据微调，训练效果很好，但是原有的预训练擅长的知识遗忘了。强化了某一部分权重，原有的均衡被打破了，知识消失

GPT3，In-context learning，and very large models

  1. prompt的原形，in-context learning：预训练之后不去动参数，通过沟通艺术的提升，让模型知道想要的结果。
  2. 不改变模型，给予模型一些示例，调整输入x，让x更符合w，那y就越接近得到想要的结果
  3. 最早的微调，提升沟通技巧

Natural Language Prompt：自然语言提示

  1. zero-shot：只给任务，不给参考答案
  2. one-shot：只给答案，不给任务
  3. few-shot：给一些参考答案

GPT3的论文发现，通过这样的方式，模型生成的质量提高了。图中三条线代表GPT1（1.3B，10亿），GPT2（13B，130亿）以及GPT3（175B，1750亿）

PEFT主流技术方案

  • 围绕token做文章：语言模型（Pre-trained Language Model，PLM）不变
    • Prompt Tuning
    • Prefix Tuning
    • P-Tuning
  • 特定场景任务：训练“本质”的低维模型
    • LoRA
    • QLoRA
    • AdaLoRA
  • 新思路：少量数据、统一框架
    • IA3
    • UniPELT

Prompt Tuning: 小模型撬动大模型，适配下游任务

  1. 让x变得更符合w，得到更想要的y。不同的问题都沿用in-context learning的套路（one-shot，zero-shot，few-shot），能否把这种“套路”学出来？
  2. w不变，改变输入的x，批量的自动化生成大模型想要的x。核心就是把原来的x加入一些前缀或者后缀，在embedding层做文章；而大模型的参数不变
  3. 图中蓝色是用了prompt design，不是自动化的设计prompt，而绿色的是自动化设计prompt design的“套路”，上面红色的是更高成本的微调

Prefix Tuning：在transformer中适配下游任务

  1. 将这种“套路”提炼到transformer中
  2. prefix是根据任务适配的，在预训练模型的前面加prefix，构成不同的transformer，通过这种组装，锁定transformer的参数，只训练prefix的参数
  3. x不变，w本身不变，在w前面加入了一些小模型权重，训练这些权重

LoRA：Low-Rank Adaptation of Large Language Model

  1. 训练w太复杂，维度太大，全部加载占用空间较大
  2. ，假设有一个 ，它由A和B相乘得到的，A和B中间有个维度是一样的 , 即要降低的rank秩，它可能只有预训练模型原有w的秩的几千几百分之一，从而将大矩阵（大模型）转化成两个小矩阵（“小模型”）
  3. 思想：在权重矩阵中，将那些重要的，高优先级的维度拎出来放到A和B中，通过训练A和B的方式实现训练完整模型的目的
  4. w不变，训练低维度的A和B
