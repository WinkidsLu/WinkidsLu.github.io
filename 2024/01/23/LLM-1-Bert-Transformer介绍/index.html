LLM (1):Bert&Transformer介绍

发表于2024-01-23|更新于2024-02-27|LLM

1. BERT(Bidirectional Encoder Repersentation from Transformers)

Google AI Language: BERT: Pre-training of Deep Bidirectional Transformers for Language Understanding

1.1 bert架构

  • bert架构: 双向transfromer-encoder架构，前后都有连接
  • GPT架构: 单向transformer-encoder架构，只向后面连接
  • ELMo架构: LSTM架构，left-to-right and right-to-left LSTM

NLP的两大任务：理解与生成
时间线：Neural Language Model(2001) -> Multitask Learning(2008) -> Word Embedding(2013) -> Neural Network for NLP(2013) -> Sequence2Sequence Model(2014) -> Attention Mechanism(2015) -> Pretrained Language(2018)
前transformer时代: RNN, LSTM, GPT
后transformer时代，统一的范式：Pre-training + Fine-tuning
数据收集预处理 -> model design -> 预训练（无标注的收集数据）-> 微调（任务相关的数据，一般需要标注，例如情感识别）

1.2 BERT的三个模块：

embedding + transformer-encoder + 预微调模块

1.2.1 embedding

分为三种embedding

- Token Embedding：字词级别的embedding
- Segment Embedding：标记这个词属于哪一句话，假设输入有多句话
- Position Embedding：标记词的位置。**与transformer不同，transformer使用三角函数标记词的绝对位置；而bert是通过学习得到的**

三种embedding通过相加得到最终的embedding representation

1.2.2 预微调模块

预微调模块，对应四种任务。对于sequence-level的分类任务，直接取[cls]对应的hidden state，后接全连接层进行分类。不同的任务是通过微调预微调模块来实现的
(1) sentence-pair classification: 预测两个句子的关联程度
(2) single sentence classification: sentiment analysis
(3) Q-A task: 例如阅读理解，输出起始到终止字符的内容作为回答内容
(4) Single sentence tagging task: NER

1.2.3 BERT的超参数选择

  • batch size: 16，32
  • learning rate: 5e-5，3e-5，2e-5
  • epoch：3，4

1.2.4 BERT的两大预训练任务

Mask LM与NSP(Next Sentence Prediction)

Mask LM：带Mask遮掩的语言训练任务

  1. 原始训练文本中，随机选择15%的token作为参与MASK任务的对象
  2. 在被选中的token中，数据生成器不是将它们全部变成[MASK]，而是按照以下三种情况处理：
    1. 80%概率下用[MASK]替换token： my dog is hairy -> my dog is [MASK]
    2. 10%概率下用一个随机单词替换token：my dog is hairy -> my dog is cat
    3. 10%概率保持token不变
  3. 模型训练过程并不知道需要预测哪些单词，在高度不确定情况下学习token的上下文语义。同时由于只有15%的token参与了mask处理，并不会破坏原本语言的表达能力

Next Sentence Prediction(NSP): 下一句话预测任务

  1. NLP中有一类问题需要模型理解两个句子之间的关系，从而需要在模型训练中引入对应任务（如QA问题，NLI-Natural Language Inference）。BERT采用的方式是输入句子对（A，B），模型预测B是不是A的真实下句话
    1. 所有参与训练的句子记作A，50%的B是原始文本中真实跟随A的下一句话（isNext，代表正样本），50%的B是原始文本中随机抽取的一句话（noNext，代表负样本）
    2. 该任务中模型可以在测试集达到97%-98%的正确率

2. Transformer

Google: Attention Is All You Need

  • bert只用了encoder
  • gpt只用了decoder
  • BART和T5用了encoder+decoder
  • 关注训练数据大小以及参数量大小（million百万，billion十亿，trillion兆）

2.1 Encoder模块

经典transformer中的Encoder模块包括6个层层堆叠Encoder Block，每个Encoder Block包括一个多头自注意力层和一个前馈全连接层（Multi-head self-attention -> feedforward）* 6

2.1.1 多头自注意力层：Multi-head Self-attention Layer

上述attention可以被描述为将query和key-value键值对的一组集合映射到输出，输出被计算为values的加权和，其中分配给每个value的权重由query与对应key的相似性函数计算得来的，这种attention被称为scaled doc-product attention，对应公式如下：

多头self-attention层则是将Q，K，V经过参数矩阵进行映射，再做self-attention，最后将结果拼接起来送入一个全连接层。

多头自注意力层的作用：实验表明，多头可以在更细致层面上提取不同head的特征，总体计算量和单一head情况下相比，提取特征的效果更佳

2.1.2 前馈全连接层

两个线性变换，中间有一个relu激活函数。

原论文中FFN的输入输出维度都是512，层内的连接维度为2048，均采用4倍的关系
前馈全连接层的作用：弥补单纯的多头自注意力机制不足以提取到理想的特征的问题，提升了网络的能力

2.2 Decoder模块

经典transformer中的Decoder模块包括6个层层堆叠Decoder Block，每个Decoder Block包括三个子层：

  • multi-head self-attention layer：与encoder模块一致，但是decoder的多头self-attention层需要额外做look-ahead-mask，在预测时不能看到未来信息，需要将当前token后面的token全部做mask
  • encoder-decoder attention layer：该层区别于自注意力机制的$Q=K=V$，此矩阵Q来源于Decoder端经过上一个Decoder Block端输出，而K和V来源于Encoder端端输出，造成$Q!=K=V$

两个注意力层的作用：多头SA层为了拟合decoder端自身信息，encoder-decoder attention 层为了整合两段的信息

  • feed-forward layer：与encoder模块完全一样

2.3 Add & Norm 模块

接在每个encoder block和decoder block中每个子层后，Add表示残差连接，Norm表示LayerNorm

  • Encoder: 多头SA -> AddNorm -> FFN -> AddNorm
  • Decoder: 多头SA -> AddNorm -> Encoder-decoder Attention -> AddNorm -> FFN -> AddNorm
  • 数学表达式：LayerNorm(x+Sublayer(x))，其中sublayer(x)为子层的输出
  • 残差连接作用：将信息传递更深，增强模型的拟合能力，增强模型的表现
  • Norm作用：随着网络层增加，通过多层计算后参数肯能出现过大过小，方差变大等现象，使学习过程出现异常，模型收敛变慢。因此对每一层计算后数值进行规范化可以提升模型的表现。
  1. BatchNorm与LayerNorm：BN是对batchsize中每个特征做归一化(不同样本相同维度特征处理成相同的分布: $(x-\mu)/\sigma$)，针对同一特征以跨样本的方式展开归一化，
  2. LN是对每个样本内的所有特征做归一化。BN竖着做，LN横着做，出发点都是为了让该层的参数稳定，缩小方差，避免梯度消失或者爆炸，方便后续学习。
  3. 如果特征依赖于不同样本之间的统计参数，BN有效，因为它抹杀了不同特征的大小关系，保留了不同样本之间的大小关系（CV）；LN抹杀了不同样本之间的大小关系，仅保留样本内不同特征的大小关系，适合NLP任务。

2.4 位置编码 Position Encoding

Transformer中直接采用正弦函数和余弦函数来编码位置信息。偶数位采用sin函数，基数为采用cos，$d_{model}$

三角函数的优势：对任意位置$PE(pos+k)$都可以表示为$PE(pos)$的线性函数，方便计算；而周期性函数不受长度限制，也能增强模型的泛化能力。

2.5 Transformers的Decoder端具体输入是什么？在训练阶段和预测阶段是否一致？

核心：

  • decoder端输入张量的特点与含义
  • decoder端在训练和预测阶段输入的形式

decoder输入张量的含义与特点

  • 6个decoder block的输入不完全相同
    • 最下面一层的block接收的输入是经过MASK后的Decoder端输入+Encoder端输出
    • 其他5层block端输入模型一致，都是前一层block端输出+encoder端端输出

Decoder在训练阶段的输入解析：

  • 2-6层输入模式一致，固定操作的循环处理
  • 第一层block：训练阶段每一个time step的输入是上一个time step的输入加上真实标签序列向后移动一位。 例如，真实标签序列为“How are you？”，当time step=1时，输入张量为特殊的token，假设为“SOS”；时间步为2时输入张量为“SOS How”，第三步为“SOS How are”，以此类推
  • 在真实代码中，训练阶段不会动态输入，而是一次性把目标序列全部速入第一层block，然后通过多头注意力机制中的MASK机制对序列进行遮掩

Decoder在预测阶段的输入解析：

  • 2-6层同理于训练阶段，无需特殊处理，固定操作的循环处理
  • 第一层block：每一步的输入都会有Encoder的输出张量，不做特殊讨论，仅专注于纯粹从Decoder端接收的输入，预测阶段每个time step的输入是从time step=0，input_tensor=“SOS”开始，直到上一个time step
    的预测输出的累计拼接张量。具体的：
    1. time=1, input=”SOS”, output=”What”
    2. time=2, input=”SOS What”, output=”is”
    3. time=3, input=”SOS What is”, output=”the”
    4. time=4, input=”SOS What is the”, output=”matter”
    5. time=5, input=”SOS What is the matter”, output=”?”
    6. time=6, input=”SOS What is the matter?”, output=”EOS”, finish decoding and perdicting.

2.6 Self-attention是什么？为什么能发挥如此大的作用？计算时如果不使用(Q,K,V)三元组而仅仅使用(Q,V)或者(K,V)或者(V)行不行？

核心

  • self-attention的机制与原理
  • 为什么要使用QKV来计算self-attention

self-attention的机制与原理

  • 一种通过自身与自身进行关联的attention机制，从而得到更好的representation来表达自身
  • self-attention是attention的一种特殊机制：$Q=K=V$，普通的attention中K与V更容易相等，序列中每个单词都会和该序列中的其他所有单词进行attention规则计算
  • attention机制的计算特点：可以直接跨越一句话中不同距离token，可以远距离学习到序列的知识依赖和语序结构

上图远距离捕捉到了语义层面的特征（its指代对象是law）。应用到传统RNN、LSTM，在获取长距离语义特征和结构特征时，需要按照序列顺序依次计算，距离越远的联系信息的损耗就越大，有效提取和捕获的可能性越小；应用self-attention时，计算过程会直接将句子任意两个token的联系通过一个计算步骤直接实现（矩阵运算）。

为什么使用(QKV)三元组？

  • 从分析的角度，查询Query是一条独立的序列信息，通过关键词Key的提示作用，得到最终语意的真实值Value表达，数学意义更充分完备
  • 为什么不用其他形式的元组，没有明确的定义，作为开放性问题
