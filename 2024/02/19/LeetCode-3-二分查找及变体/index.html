LeetCode(3):二分查找及变体

发表于2024-02-19|更新于2024-02-20|LeetCode
  • 二分查找 - 红蓝染色法
  • 二分查找的三种写法：闭区间、半开半闭区间、开区间
  • 处理>=,>,<,<=四种情况

题目：

  • 34. 在排序数组中查找元素的第一个和最后一个位置（三种写法）
  • 162. 寻找峰值
  • 153. 寻找旋转排序数组中的最小值
  • 33. 搜索旋转排序数组(两种方法)

No.34 在排序数组中查找元素的第一个和最后一个位置（Normal）

给你一个按照非递减顺序排列的整数数组nums，和一个目标值target。请你找出给定目标值在数组中的开始位置和结束位置。如果数组中不存在目标值target，返回[-1, -1]。你必须设计并实现时间复杂度为$O(\log n)$的算法解决此问题。

示例 1：
输入：nums = [5,7,7,8,8,10], target = 8
输出：[3,4]

示例 2：
输入：nums = [5,7,7,8,8,10], target = 6
输出：[-1,-1]

示例 3：
输入：nums = [], target = 0
输出：[-1,-1]

解答：闭区间写法

利用数组有序的性质，假设判断条件是>=8，取中间，如果它小于target，则他左边的数都小于target，左边都确定了，于是将左端点移动为left=mid+1；如果中间>=8，则他右边的都确定了，更新right=mid-1。

终止条件：left>right，左指针跑到了右指针的左边，此时的左指针为返回结果

四种情况：

  • >x -> >=x+1 (整数)
  • \ >= x左边那个数

时间复杂度：$O(\log n)$

No.162 寻找峰值

峰值元素是指其值严格大于左右相邻值的元素。给你一个整数数组nums，找到峰值元素并返回其索引。数组可能包含多个峰值，在这种情况下，返回任何一个峰值所在位置即可。
你可以假设 nums[-1] = nums[n] = -∞。
你必须实现时间复杂度为$O(\log n)$的算法来解决此问题。

解答：红蓝染色法

  • 红色为false，代表峰值左侧的元素
  • 蓝色为true，代表峰值或者峰值右侧的元素
  • 由于题中说峰值肯定存在，那么最右侧的元素肯定为峰值或峰值右侧，即一定是蓝色的，那么设置左指针left=0，right=n-2
  • 通过比较M和M+1的大小进行染色，如果M小于M+1，则M一定在封顶左侧，左边都是红色；如果M大于M+1，那么M要么是封顶，要么在封顶右侧

No.153 寻找旋转排序数组中的最小值

已知一个长度为n的数组，预先按照升序排列，经由1到n次旋转后，得到输入数组。例如，原数组 nums = [0,1,2,4,5,6,7] 在变化后可能得到：
若旋转4次，则可以得到 [4,5,6,7,0,1,2]
若旋转7次，则可以得到 [0,1,2,4,5,6,7]
注意，数组[a[0], a[1], a[2], ..., a[n-1]]旋转一次的结果为数组[a[n-1], a[0], a[1], a[2], ..., a[n-2]]。

给你一个元素值互不相同的数组nums，它原来是一个升序排列的数组，并按上述情形进行了多次旋转。请你找出并返回数组中的最小元素 。

你必须设计一个时间复杂度为$O(\log n)$的算法解决此问题。

解答：红蓝染色法。

  • 红色false：最小值左侧
  • 蓝色true：最小值或者最小值右侧
  • 可以确定最后一个数，要么是最小值，要么在最小值的右侧，即肯定为蓝色，所以在0到n-2的范围内进行二分
    • 与最后一个数比大小，mid如果小于最后一个数，存在两种情况：在一段递增序列中(mid在最小值右侧)或者在两段递增序列的第二段。无论是哪种情况，mid要么为最小值，要么在最小值的右侧，染成蓝色
    • mid如果大于最后一个数，一定存在于两段递增序列中，那么他肯定在最小值的左侧（大于最后一个数，最后一个数是二段递增序列里最大的），染成红色

No.33 搜索旋转排序数组(两种方法)

整数数组nums按升序排列，数组中的值互不相同。在传递给函数之前，nums 在预先未知的某个下标k（0 <= k < nums.length）上进行了旋转，使数组变为 [nums[k], nums[k+1], ..., nums[n-1], nums[0], nums[1], ..., nums[k-1]]（下标从0开始计数）。例如， [0,1,2,4,5,6,7]在下标3处经旋转后可能变为[4,5,6,7,0,1,2]。
给你旋转后的数组nums和一个整数target，如果nums中存在这个目标值target ，则返回它的下标，否则返回-1 。
你必须设计一个时间复杂度为 O(log n) 的算法解决此问题。

示例 1：
输入：nums = [4,5,6,7,0,1,2], target = 0
输出：4

示例 2：
输入：nums = [4,5,6,7,0,1,2], target = 3
输出：-1

示例 3：
输入：nums = [1], target = 0
输出：-1
