LLM(0):大模型导论 —— 技术前沿

发表于2024-02-29|更新于2024-03-25|LLM

大语言模型技术前沿

1. 简介

  • 从Pre-train LM到Large LM：
  1. PLM基于Transformer，采用pre-train+fine-tune的训练方式
  2. PLM通过自监督的方式得到，不需要人工标注（数据质量高，模型算力要求高），也可以通过supervised fine-tuning（SFT）的方式让模型与人类的需求对齐
  3. LLM：Instruction tuning指令微调，对齐RLHF
  • 主流开源大语言模型：
    • LLaMA 1T
    • LLaMA2 2T
    • ChatGLM 1T
    • ChatGLM2 1.4T
    • Falcon 1.5T
    • Bloom 350B
    • Baichuan 1.2T
  • 大模型文件：模型参数文件(parameters) + 执行代码文件(run.c)
  • 大模型研究方向：Training、Inference、Evaluation、Applications、Challenges
  • 大模型的训练流程
    1. 预训练：base模型（基座模型），这里得到的是预测下一个词的模型，当问一个问题，可能会得到更多的问题。该阶段模型不具备领域适用
    2. Instruction Tuning指令微调：让大模型服务人类任务，模型对齐
    3. 指令微调时会引入强化学习方法，叫做Reinforcement Learning from Human Feedback（RHLF），或者叫做Human Preference（人类偏好）。这里使用OOP技术：当模型做了一定程度微调后，效果没有那么好，让模型输出两个句子，让人工去判断两个句子哪个好，给予模型人类反馈，让模型学习一个reward model去调整模型的参数。

2. 模型架构

  • Encoder + Decoder：T5，Flan-T5，BART
  • Casual Decoder（主流）：GPT，LLaMA，BLOOM，OPT。从第一个token开始，每次预测一个新的词。训练效率高
  • Prefix Decoder：GLM，ChatGLM，U-PaLM。输入的时候已经有了几个词，通过已有的词去预测下一个词。训练效率低

3. Tokenization（效率和性能之间的平衡）

  • Byte-Pair Encoding（BPE）：character-level，寻找临近的两个字符出现概率最大的，合并成新的字符，不断的根据概率组合新的字符。ChatGPT和大部分主流大模型使用的是BPE
  • WordPiece：拆成字符并合并，合并标准中BPE用的是他们出现的频次，而WordPiece训练了N-gram模型，让模型概率最高作为合并的依据。BERT使用WordPiece
  • UnigramLM
  • SentencePiece

4. Positional Encoding

  • 绝对位置编码：token在句子中具体是第几个
  • 相对位置编码：不知道token在句子中位置是第几个，只知道token距离别的token有多远
    • Attention with Linear Biases（ALiBi）：绝对位置编码中输入模型的embedding是token embedding和positional embedding的加和。而ALiBi不是简单的相加，而是在transformer结构attention的操作上注入了位置信息
    • RoPE旋转位置编码

5. Attention Mechanism

  • self-attention
  • cross attention
  • full attention
  • sparse attention: sparse加上部分全局，较少的降低模型表现
  • flash attention：内存机制上的创新
  • multi-query attention

6. Activation

  • ReLU：max函数，方法简单效果好，为神经网络增加了非线性的操作
  • Gaussian Error Linear Unit（GeLU）：ReLU+Dropout+Zoneout，目前在大模型中应用较多
  • Gate Linear Unit（GLU）

7. 层归一化（稳定训练，分散分布变集中）

  • LayerNorm
  • RMSNorm
  • Pre-Norm，Post-Norm，Sandwhich：post-norm在attention之后加的layernorm，post-norm在attention之前加的，sandwhich在attention两边各加的layernorm，目前sandwhich用的最多
  • DeepNorm

8. Augmented LLMs

问题：知识库的更新（模型基于某年前的知识库）、模型序列长度容纳不下（1024）

  • Retrieval Augmented LLMs（RAG）：外挂向量数据库，数据库中有许多知识。防止模型输入胡乱的答案，用户问题在输入模型之前在数据库中作similarity的检索，找到其中某些段落，高相关的段落抽出来加到user input的前面。它不是模型本身的架构
  • Tool Augmented LLMs

9. 上下文长度

支持token长度：

  • LLaMA:2048-4096-32k
  • GPT3.5:4096
  • GPT4:128k(300 pages PDF)

10. 推理加速技巧

-量化（Quantization）：降低模型参数精度以减少参数量大小，例如float32 to int8(int4)

  • 剪枝（Pruning）
  • 蒸馏（Distillation）
  • 参数共享（weight sharing）
  • 矩阵分解（Factorization）

11. 数据集和基线（大模型的测评）

  • MMLU：多任务语言理解（包括STEM和社科之类的内容），形式为选择题。
  • SuperGLUE：传统NLP任务，如机器翻译，阅读理解
  • BIG-Bench BBH：开源自贡献数据集
  • GSM8K：数学题，一般回答形式是problem-solution-final answer，最后根据答案判定是否做对。
  • GLUE
  • HumanEval：代码生成，包含功能描述和输入输出样例。这里的评价指标包含pass@1和pass@x，pass@1表示根据greedy生成的通过率，pass@x代表多写几次去测，当然pass@x的效果还是要高很多的……
  • AGIEval
  • GLORE：逻辑推理

https://zhuanlan.zhihu.com/p/669193585

  • GPT4, PaLM-E, 预训练和微调流程，DeepSpeed分布式训练工具库
  • 大模型推理相关算法，如微调（Prefix-tuning，P-Tuning V1，LoRA，P-Tuning V2），RLHF，可控算法，并行训练，量化。
  • 主流大语言模型：llama，miniGPT，chatglm-6b，baichuan-7b
    掌握最先进的AI模型算法，并对大模型的训练和调优（P-tunning，LoRA技术等）有丰富经验，如：GLM-2系列、Llama-2系列、GPT系列、熟悉常见的模型结构，有成功应用经验或者独立实现过算法创新者优先；
  1. 熟悉大模型压缩，模型加速，模型蒸馏技术，握基本理论以及常用工具者优先。
    熟悉GPT、llama、Stable Diffusion、Midjourney、DALLE等生产式AI领域重要技术解决方案；
    熟悉A100、V100、A30、T4等GPU服务器的特点，并掌握其显卡驱动、docker环境配置，了解AI项目对显卡和基础系统软件的要求；
    5、对prompt工程、参数调优工程、模型精调工程有深入的理解和实践能力；
