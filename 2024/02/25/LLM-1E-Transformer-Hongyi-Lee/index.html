Hung-yi Lee Transformer

a set of sequences -> a set of labels: NER, POS词性标注
a set of sequences -> a label: sentiment analysis
a set of sequences -> model decide the number of output itself: Seq2Seq

(Multi-head) Self-attention

如何解决上面这种情况？例如在词性标注任务中，fc无法区分出两个saw的真正差别，需要让模型考虑上下文的信息？可以给一个window，让fc考虑上下文neighbor的信息，这种做法仍然有瑕疵。

self-attention（Attention is all you need）

对于一串vector，可以是input，或者是某个hidden layer的输出，self-attention也输出一串vector，但是每个vector都考虑了整个序列的信息。

以b1为例子，生成过程：

  1. 在a中找出与a1相关的向量，即对a1起决定作用的向量，用 $\alpha$ 表示。如何确定这种相关程度并给出数值？计算attention的模组，计算两个向量的 $\alpha$，常用做法是doc-product：两个向量分别乘上两个矩阵，得到q和k，再将qk进行doc-product得到 $\alpha$

具体的计算过程为：a1分别与自己，与序列中的其他vector计算attention-score: $\alpha$，将所得到的$\alpha$进行softmax处理(不一定要用softmax，只是原论文中使用了softmax), 接下来，根据得到的attention score $\alpha$，抽取序列中重要的信息：a1到a4乘以矩阵 $W^v$，分别得到v1,v2,v3,v4，最终b1的结果是通过 $\alpha$ 和 $v$ 相乘相加得到。

这样，通过一系列input vector就能得到一系列output vector。注意b1到b4是同时计算出来的。

b2的计算和b1，类似，首先根据q2和k1，k2，k3，k4分别做doc-prodcut得到attention score，然后经过归一化，例如softmax，然后与v做doc-production。

从矩阵乘法的角度介绍self-attention的运作原理

$Q=W^Q\cdot I$, $K=W^K\cdot I$, $V=W^V\cdot I$

总结：self-attention就是一系列的矩阵运算，其中需要学习的参数就是三个w矩阵：$W^q$, $W^k$, $W^v$

进阶版本：Multi-head self-attention（head的数目作为hyperparameter，需要根据具体任务调整）
通俗的解释多头：self-attention时用q去找相关的k，但是“相关”有多种不同的形式或者定义，因此需要有不同的q。
多头的实现：a乘上Wq得到q，q再乘上一个不一样的矩阵，如图一个头以及另外一个头的例子（1一起做，2一起做）

得到多个bij，接起来再通过一个矩阵transform得到最终的bi。

Position Encoding

self-attention中没有位置信息，如何考虑到位置这种重要的信息？使用位置编码技术，为每个位置设定一个不同的位置向量$e_i$，这样的vector时hand-crafted人为设定的，在论文中使用位置编码sin和cos产生的。（没有严格定义，位置编码可以自由创作，也可以通过数据习得）

Transformers 模型

Transformer is a Sequence-to-Sequence(Seq2Seq) model, input is seq and output is seq, the length of output is determined by model.

Encoder

输入 ${x_1,x_2,x_3,x_4}$，输出 ${h_1,h_2,h_3,h_4}$，encoder中有很多block，每个block中做的事是好几层做的事

  1. 输入vector
  2. self-attention，残差residual连接，layernorm（不用考虑batch的信息）
  3. fc层，残差连接，layernorm。

Decoder (Autoregressive Decoder, AT Decoder)

Decoder在每个时间段都输入是前一个时间段它自己的输入，他会将自己的输出当作接下来的输入。

Decoder与Encoder对比：

  1. Decoder是masked multi-head self-attention（不加masked时，生成b1的时候考虑a1-a4的信息，然而在masked中生成b1时只考虑a1，生成b2只考虑a1和a2，生成b3只考虑a1-a3，而不考虑后面部分的信息）；ji在计算b2时，只用q2和k1，k2分别做点积，而不考虑后面的k3和k4 —— 为什么mask？输出是顺序产生的，每次输出只能考虑左边的信息而不能考虑右边的信息。

Decoder端需要自己决定输出的序列长度，无法直接判断输出的序列长度

encoder与decoder之间传递信息：cross attention，q来自decoder，k和v来自encoder

关于cross attention：

  1. 原始paper中的multi-head self-attention，decoder每一层的cross attention都是用encoder最后一层输出的k，v来参与运算。实际结果上不一定这样，有论文尝试了不同层的cross attention

Training

  1. 输出的distribution和对应正确答案的one-hot vector越接近越好，每一个token都和真实结果one-hot向量作cross-entropy，最终希望总和的cross-entropy越小越好。
  2. 训练时，decoder输入的是正确答案，using the ground truth as input（Teacher Forcing）
  3. 训练技巧
    1. Copy Mechanism：decoder没必要每次都自己输出，还可以在输入中复制一些东西进行输出。例如：你好库洛洛，输出：库洛洛你好，很高兴认识你，这里的库洛洛不是生成的，而是从输入中复制出来的。这个技巧适用于对话系统或摘要生成
    2. Guided Attention：要求机器在attention时具有固定的方式。例如语音辨识，语音合成要求固定的由左向右方向
    3. Beam Search：
  4. Minimize cross-entropy in training 不一定会 maximum BLUE scores。训练时看的cross-entropy，而最终作模型选择时会根据句子与句子的BLUE scores。在训练时将BLUE scores作为优化目标不太容易，其不可微分，如何？ —— 遇到optimize无法解决的问题，尝试使用Reinforcement Learning，将decoder作为agent，将BLUE scores作为reward（难度高）
  5. 训练时decoder看到完全正确的输出，而测试时看到的是自己的输出，这不一致的现象叫做exposure bias，可以解决的方法是在训练的时候给decoder一些错的东西（scheduled sampling）

https://www.zhihu.com/question/362131975/answer/3360076979
在seq2seq任务中，Transformer encoder显然需要和decoder联合训练，因为这不是个分类问题，encoder没法单独接MLP进行训练。其中，token流经encoder的过程比较简单，用self-attention就能解决，难点是这个decoder。首先要思考一个问题，如果让你设计如何使用这个decoder，你可以做到让decoder一次性得到所有的输出token吗？答案是不可能。一个序列的生成必须有一个结束的token（如[EOS]）来提示，而这个结束的token的生成必须依赖前面已经生成的token，因为打算结束一个句子，必须是因为这个句子已经完成了任务；也就是说，最后一个token的生成必须由前面生成的token来condition。以此类推，前面的每一个token都必须由更前面的token来condition，直到推到起始token（如[SOS]）。这和人说话的逻辑是相似的，人也许可以在脑中构思整个句子，但是表述一定是一个词一个词说出来的，而且后面的词一定会被前面的词所影响，这就是说话的逻辑。因此，decoder在预测的时候依然是一个一个token去预测的，不可能一次性全部输出。这个逐个token输出的过程如下图所示。也就是说，decoder的预测算法无法并行。但是decoder的训练采用了一个不同的、很精妙的方式，叫做teacher forcing，也就是有老师带着做训练。这其实不是Transformer的创新点，在那个时代已经是一个老生常谈的话题了。teacher forcing是什么意思呢？我们拿RNN decoder的teacher forcing举一个例子。如下图所示，我们假设一个seq2seq的输入是What do you see，正确的输出(label)是Two people running，那么下图左边的训练过程就叫做free-running方法，下图右边的就叫做teacher forcing方法。如果你仔细观察Transformer结构图的右下角，会发现有一个提示是把Outputs做一个Shifted Right，也就是将所有输入token都往右移一个位置。这就对应着下图中右边所示的teacher forcing方法，是把label的所有token向右移一个位置，然后在最左边放上一个表示开始的token（例子中是）。这种方法和free-running有什么区别呢？很显然，free-running的decoder RNN的输入是上一时刻自己输出的值，而这不一定是正确的；而teacher forcing的decoder RNN的输入是上一时刻的label，这一定是正确的。这种训练方法可以防止错误的累积，从而提高训练效果。
